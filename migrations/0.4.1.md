# Migration 0.4.1

Applied by `job-tracker:health` when `config/.migrated-version` < `0.4.1`.

Each step is idempotent: check first, write only if the marker is absent.
Stop on first failure; do not update `config/.migrated-version` until all steps pass.

---

## Step 1 — Rename legacy PDF CSS path from `resume.css` to `cv.css`

**Target files:** `scripts/resume.css`, `scripts/cv.css`, and `config/paths.md`.

**Check:** this step is satisfied only when both conditions are true:
- `scripts/cv.css` exists.
- `config/paths.md` does not contain `scripts/resume.css`.

If both conditions are true -> skip.

**Apply:**

1. If `scripts/cv.css` is missing and `scripts/resume.css` exists, move `scripts/resume.css` to `scripts/cv.css`.
   - Preserve the file content exactly.
   - Do not copy the canonical package CSS over a user-customized legacy CSS file.
2. If both `scripts/cv.css` and `scripts/resume.css` exist, leave `scripts/cv.css` unchanged and remove `scripts/resume.css`.
3. If both `scripts/cv.css` and `scripts/resume.css` are missing:
   - In plugin mode, copy `$CLAUDE_PLUGIN_ROOT/scripts/cv.css` to `scripts/cv.css` if that source exists.
   - In workspace mode, this step is not satisfied; stop and report `warning: scripts/cv.css is missing and legacy scripts/resume.css is unavailable`.
4. In `config/paths.md`, replace every exact string `scripts/resume.css` with `scripts/cv.css`.

**Success condition:** `scripts/cv.css` exists and `config/paths.md` contains `scripts/cv.css` and does not contain `scripts/resume.css`.

**Failure condition:** `scripts/cv.css` is still missing, `config/paths.md` is missing, or `config/paths.md` still contains `scripts/resume.css` after the write. Stop and report. Do not bump `.migrated-version`.
