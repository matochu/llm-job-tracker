# Migration 0.3.1

Applied by `job-tracker:health` when `config/.migrated-version` < `0.3.1`.

Each step is idempotent: check first, write only if the marker is absent.
Stop on first failure; do not update `config/.migrated-version` until all steps pass.

---

## Step 1 — Add `## Fit Score Bands` to search profiles

**Target files:** every `strategy/search-profiles/*.md`.

**Check:** search each file for the exact string `## Fit Score Bands`.
- If found → step is satisfied for that file; skip it.
- If not found → apply.

**Apply:** append the following block verbatim at the end of the file.
Ensure exactly one blank line between the last existing content and the new heading.

```
## Fit Score Bands

Adjust thresholds to match your search strategy. These are conservative defaults.

- **Strong apply (≥45/60):** apply immediately, no tailoring needed.
- **Apply with tailoring (35–44/60):** worth applying; address top gaps in CV before sending.
- **Low ROI (<35/60):** skip unless strong referral or very low competition.
```

**Success condition:** the string `## Fit Score Bands` is present in the file after the write.

**Failure condition:** the write failed, or the string is still absent after the write. Stop and report the file path and error.

---

## Step 2 — Add `candidate/application-answers.md` to `config/paths.md`

**Target file:** `config/paths.md`.

**Check:** search the file for the exact string `application-answers.md`.
- If found → step is satisfied; skip.
- If not found → apply both sub-steps below.

### Sub-step 2a — Candidate zone bullet

**Anchor:** find the line that begins with `- **Candidate (`:

```
- **Candidate (`candidate/`):**
```

Insert the following as a new bullet on the line immediately after the anchor line:

```
- **Application answer bank:** `candidate/application-answers.md`
```

**Success condition:** the string `Application answer bank` is present in the file after the write.

**Failure condition:** the anchor line is not found, or the string is absent after the write. Stop and report.

### Sub-step 2b — Runtime Artifacts entry

**Anchor:** find the line that begins with `- **Story bank:**`:

```
- **Story bank:** `candidate/stories.md`
```

Insert the following as a new line immediately after the anchor line:

```
- **Application answer bank:** `candidate/application-answers.md`
```

**Success condition:** `candidate/application-answers.md` appears in the `## Runtime Artifacts` section after the write (or anywhere in the file if that section heading is absent).

**Failure condition:** the anchor line is not found, or the string is absent after the write. Stop and report.

## Step 3 — Seed `candidate/application-answers.md` (plugin mode only)

**Check:** if `candidate/application-answers.md` exists in the workspace root → step is satisfied; skip.

**Mode guard:** if `CLAUDE_PLUGIN_ROOT` is not set (workspace mode) → step is not applicable; skip.
In workspace mode the CLI already seeds this file; treat as satisfied.

**Apply (plugin mode only):**
Copy `$CLAUDE_PLUGIN_ROOT/candidate/application-answers.md` to `candidate/application-answers.md` in the workspace root.

**Success condition:** `candidate/application-answers.md` exists and is non-empty after the copy.

**Failure condition:** `$CLAUDE_PLUGIN_ROOT/candidate/application-answers.md` does not exist or cannot be read. This step is **not satisfied** — do not count it as complete. Report as `warning: seed source missing at $CLAUDE_PLUGIN_ROOT/candidate/application-answers.md`. Do not bump `config/.migrated-version`.
